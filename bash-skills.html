Navigation
⌘ pwd Print working directory ▾
What it does
Shows your exact location in the filesystem as a full path.
Example
$ pwd /home/anazial/.openclaw/workspace
Tip
Use it whenever you feel lost. Think of it as "You Are Here" on a map.
⌘ ls List directory contents ▾
What it does
Shows all files and folders in your current directory.
Example
$ ls AGENTS.md BOOTSTRAP.md README.md
Common flags
-a -l -h -lah
⌘ cd Change directory ▾
What it does
Moves you into a different folder.
Examples
$ cd documents $ cd .. $ cd ~/.openclaw/workspace $ cd /var/www/html
Tip
cd .. goes up one level. cd ~ takes you home from anywhere.
⌘ hostname Show machine name ▾
What it does
Prints the name of the machine your shell is running on.
Example
$ hostname enid
Tip
Use it as a sanity check when jumping between your Mac and Enid via SSH.
⌘ whoami Show current user ▾
What it does
Prints the username you are currently logged in as.
Example
$ whoami anazial
Tip
Useful after sudo su to confirm whether you are root or your regular user.
Files & directories
⌘ cat Print file contents ▾
What it does
Dumps the entire file to the terminal at once. Best for short files.
Example
$ cat AGENTS.md
Tip
If your screen floods with text, switch to less instead.
⌘ less Scrollable file viewer ▾
What it does
Opens a file in a scrollable viewer. Better than cat for long files.
Example
$ less BOOTSTRAP.md
Controls
Arrow keys or spacebar to scroll. q to quit.
⌘ head Show first 10 lines ▾
What it does
Prints just the top of a file. Good for peeking.
Example
$ head AGENTS.md $ head -20 AGENTS.md
Common flags
-n
⌘ tail Show last 10 lines ▾
What it does
Prints the bottom of a file. Especially useful for log files.
Example
$ tail error.log $ tail -20 error.log
Common flags
-n
⌘ wc Word / line count ▾
What it does
Counts lines, words, or characters in a file.
Example
$ wc -lw AGENTS.md 42 310 AGENTS.md
Common flags
-l -w
⌘ file Identify file type ▾
What it does
Tells you what kind of file something is -- useful when the extension is missing or unclear.
Example
$ file image.png image.png: PNG image data, 1920 x 1080
⌘ find Search for files ▾
What it does
Searches for files by name, type, or other attributes. Searches recursively through subdirectories.
Examples
$ find ~ -name "AGENTS.md" $ find . -name "*.md" $ find / -name "nginx.conf"
With -exec
Run a command on every result. {} is the placeholder for each file found.
$ find . -name "*.rtf" -exec pandoc {} -o {}.md \;
Common flags
-name -exec -type
⌘ grep Search inside files ▾
What it does
Searches for text inside files. find looks for files, grep looks inside them.
Example
$ grep "session" AGENTS.md $ grep -r "session" ~/documents
Common flags
-r -i -n
⌘ mv Move or rename ▾
What it does
Moves a file or directory to a new location. Also used to rename things.
Examples
$ mv file.md ~/documents/ $ mv oldname.md newname.md $ mv folder/ ~/documents/ $ mv file.md .
Tip
Does not need -r for directories. The original is gone after moving.
⌘ cp Copy files ▾
What it does
Copies a file or directory. Original stays in place.
Examples
$ cp file.md ~/documents/ $ cp -r myfolder/ ~/documents/ $ cp *.md ~/documents/
Common flags
-r
⌘ rm Delete files ▾
What it does
Permanently deletes files or directories. No trash, no undo.
Examples
$ rm file.md $ rm -r myfolder/ $ rm -ri folder1/ folder2/
Common flags
-r -f -i
Warning
-rf deletes everything forcefully with zero confirmation. Double check before running.
⌘ mkdir Create directory ▾
What it does
Creates a new empty directory.
Example
$ mkdir newfolder $ mkdir ~/documents/projects
⌘ du Disk usage ▾
What it does
Shows how much disk space a directory is using.
Examples
$ du -sh myfolder/ $ du -sh * $ du -sh .
Common flags
-s -h
⌘ stat Read file metadata ▾
What it does
Reads everything the filesystem knows about a file -- permissions, size, owner, timestamps, inode number. Works on files, directories, symlinks, and devices.
Default output
$ stat myfile.md File: myfile.md Size: 1234 Blocks: 8 regular file Access: (0644/-rw-r--r--) Uid: (1000/anazial) Modify: 2026-05-20 09:00:00
Custom format (GNU / Ubuntu)
Use -c with format specifiers to print only what you need.
$ stat -c "%a %n" * 644 readme.md 755 myscript.sh $ stat -c "%a %A %U %G %n" myfile.md 644 -rw-r--r-- anazial anazial myfile.md
macOS difference
macOS uses BSD stat. Use -f instead of -c, and %Lp for octal permissions.
$ stat -f "%Lp %N" * # macOS only
Common flags
-c
⌘ pandoc Convert file formats ▾
What it does
Converts between document formats. Figures out the conversion from file extensions.
Examples
$ pandoc file.rtf -o file.md $ pandoc file.md -o file.docx
Batch convert with loop
$ for f in *.rtf; do pandoc "$f" -o "${f%.rtf}.md" done
Common flags
-o
Remote & SSH
⌘ ssh Connect to remote machine ▾
What it does
Opens a secure shell session on a remote machine. Everything you type runs there, not locally.
Examples
$ ssh anazial@100.x.x.x $ ssh anazial@enid
Tip
With Tailscale you can use the machine hostname instead of the IP address.
⌘ exit Leave SSH / switch user ▾
What it does
Ends the current shell session. Drops you back to your local terminal after SSH, or back to your regular user after sudo su.
Example
$ exit
Force disconnect
If the terminal is frozen, type ~. (tilde then period) to force close SSH.
⌘ scp Secure copy between machines ▾
What it does
Copies files between your local machine and a remote one over SSH. Run from your local machine.
Examples
$ scp file.md user@enid:~/ $ scp user@enid:~/file.md ~/Desktop/ $ scp -r folder/ user@enid:~/ $ scp user@enid:~/folder/* ~/Desktop/
Pattern
Remote paths use user@host:/path format. The colon separates machine from path.
Common flags
-r
⌘ tailscale status List Tailscale devices ▾
What it does
Shows all devices on your private Tailscale network with their IPs and hostnames.
Example
$ tailscale status
System & admin
⌘ sudo Run as administrator ▾
What it does
Superuser do. Temporarily runs one command with full admin privileges.
Examples
$ sudo apt install nginx $ sudo mv file.html /var/www/html/ $ sudo reboot
Tip
sudo su switches you fully into the root user. Use exit to get back out.
⌘ systemctl Manage background services ▾
What it does
Starts, stops, restarts, and checks the status of system services on Ubuntu.
Examples
$ sudo systemctl status nginx $ sudo systemctl start nginx $ sudo systemctl stop nginx $ sudo systemctl restart nginx
⌘ apt Install software (Ubuntu) ▾
What it does
Ubuntu's package manager. Installs, updates, and removes software.
Examples
$ sudo apt install nginx $ sudo apt update $ sudo apt update && sudo apt upgrade nginx
⌘ brew Install software (Mac) ▾
What it does
Homebrew -- Mac's package manager. Equivalent of apt on Ubuntu.
Examples
$ brew install helix $ brew install pandoc $ brew services start nginx
⌘ echo $SHELL Show current shell ▾
What it does
Prints which shell you are using. echo prints a value, $SHELL is an environment variable holding the shell path.
Example
$ echo $SHELL /bin/zsh (Mac) /bin/bash (Ubuntu)
⌘ sudo reboot Restart the machine ▾
What it does
Restarts the machine. If you are SSH'd in, your connection will drop immediately -- that is normal.
Example
$ sudo reboot
Tip
Wait 30 seconds then SSH back in as normal.

Reusable syntax elements that work across many commands. Click to expand.

Path symbols
~
Home directory
▾
What it means
A shortcut for your home directory. Works anywhere a path is expected.
Examples
$ cd ~ $ ls ~/documents $ scp file.md user@enid:~/
.
Current directory
▾
What it means
Refers to wherever you currently are. Useful as a destination.
Examples
$ mv ~/downloads/file.md . $ cp ~/documents/* . $ du -sh .
..
Parent directory
▾
What it means
One level up from where you are.
Examples
$ cd .. $ cd ../../ $ mv file.md ../
/
Root / path separator
▾
What it means
At the start of a path, / means the absolute root of the filesystem. Between path parts, it separates directory names.
Examples
$ cd /var/www/html $ ls / $ find / -name "nginx.conf"
Tip
Paths starting with / are absolute (always from root). Paths without / are relative (from where you are).
Wildcards & patterns
*
Wildcard -- match anything
▾
What it means
Matches any characters. Use it to target groups of files by pattern.
Examples
$ ls *.md $ cp *.txt ~/backup/ $ scp user@enid:~/folder/* ~/Desktop/ $ find . -name "*.rtf"
Contents vs directory
folder/ copies the folder itself. folder/* copies what is inside it.
Operators & chaining
;
Run sequentially
▾
What it does
Runs commands one after the other, regardless of whether the first one succeeded.
Example
$ mkdir folder; cd folder
&&
Run if previous succeeded
▾
What it does
Only runs the second command if the first one succeeded. Safer than semicolon for dependent commands.
Example
$ sudo apt update && sudo apt upgrade nginx
Tip
If apt update fails, the upgrade won't run. Good for safety.
|
Pipe -- chain commands
▾
What it does
Takes the output of one command and feeds it as input to the next.
Example
$ cat file.txt | grep "session" $ ls | wc -l
Tip
The second example counts how many files are in a directory.
$()
Command substitution
▾
What it does
Runs a command inside the parentheses and uses its output as a value in another command.
Example
$ for f in $(find . -name "*.txt"); do pandoc "$f" -o "${f%.txt}.md" done
Tip
The find command runs first, then its list of files is used by the for loop.
> and >>
Redirect output to file
▾
What it does
Instead of printing to the terminal, sends output to a file.
Example
$ echo "hello" > file.txt $ cat log.txt >> archive.txt
Difference
> overwrites the file. >> appends to the end without overwriting.
\
Escape -- handle spaces
▾
What it does
Escapes the next character. Most commonly used to handle spaces in file or directory names.
Example
$ cd My\ Documents $ mv file.md 00\ Human/001\ To\ Be\ Sorted/
Tip
Alternatively, wrap the whole path in quotes: cd "My Documents"
Scripting
for loop
Repeat for each item
▾
What it does
Repeats a command for each item in a list. The variable (f) takes the value of each item in turn.
Pattern
for f in *.rtf; do pandoc "$f" -o "${f%.rtf}.md" done
Breaking it down
for f in *.rtf -- loop through every rtf file, calling it f each time. do...done -- wraps the commands to run on each. ${f%.rtf} -- strips the .rtf extension from the filename.

The bigger ideas behind the commands. Click to expand.

CLI
Command Line Interface
▾
A text-based way to interact with a computer. You type commands, the computer responds with text. The alternative is a GUI (Graphical User Interface) where you click around. The CLI feels less intuitive at first but is far more powerful and scriptable.
Shell
bash vs zsh
▾
The shell is the program that reads your commands and runs them. bash is the default on Ubuntu, zsh is the default on modern macOS. They are almost identical for everyday use -- the differences only matter in advanced scripting.
Mac prompt: % Ubuntu prompt: $ Root prompt: #
Absolute vs relative paths
Where you start from matters
▾
An absolute path starts from the root of the filesystem and always works from anywhere. A relative path starts from wherever you currently are.
Absolute: /var/www/html/index.html Relative: documents/index.html
sudo & root
Admin privileges
▾
Root is the all-powerful admin account on Linux -- it can do anything, including break things. sudo (superuser do) lets your regular user borrow root power for a single command. sudo su switches you fully into root -- your prompt changes from $ to #. Always exit back out when done.
File permissions
Who can do what
▾
Every file has permissions for three groups: owner, group, and others. Each group can have read (r), write (w), and execute (x) rights. These appear symbolically in ls -l or as octal numbers with stat.
-rw-r--r-- → 644 drwxr-xr-x → 755 -rw------- → 600 (private)
The first character is the file type: - for file, d for directory, l for symlink.
Hidden files
Dotfiles
▾
Any file or directory starting with . is hidden from ls by default. Configuration files are often hidden this way. Use ls -a to see them.
~/.openclaw ~/.config ~/.gitconfig
Package managers
brew and apt
▾
Package managers install, update, and remove software from a central repository. Think of them as app stores for the terminal. brew on Mac, apt on Ubuntu.
Mac: brew install helix Ubuntu: sudo apt install helix
Git & .git
Version control
▾
Git is a version control system -- a time machine for your files. It tracks every change so you can go back in time. The .git directory is a hidden folder Git creates inside any repository to store this history. It contains write-protected files which is why rm -r asks for confirmation.
nginx
Web server
▾
nginx is a web server -- a program that listens for browser requests and serves files in response. It runs as a background service managed by systemctl. By default it serves files from /var/www/html/ and index.html loads automatically when you visit the root URL.
LTS
Long Term Support
▾
LTS versions of software are supported with security updates for a longer period (5 years for Ubuntu). They are more stable and predictable than cutting-edge releases. Ubuntu 24.04 LTS (Noble Numbat) is what Enid runs.
.bak convention
Keeping backups
▾
Adding .bak to a filename is a common convention meaning "backup". The file is still there if you need it but programs and servers won't pick it up automatically. Better than deleting something you might want later.
$ mv index.nginx-debian.html index.nginx-debian.html.bak
Tailscale & SSH
Private networking
▾
SSH creates an encrypted tunnel to a remote machine. Tailscale creates a private network between your devices so you can SSH using hostnames instead of IPs, and services like nginx are only reachable by your tailnet devices -- not the public internet.
Markdown
.md files
▾
Markdown is a simple text formatting language. Files end in .md. # makes a heading, ** makes bold, - makes a bullet. It is human-readable as plain text and also renders nicely in many apps. Commonly used for documentation and README files.
Symlinks
Symbolic links
▾
A symlink is a file that points to another file or directory -- like a shortcut. The OS transparently redirects you to the target when you open it. Shows up as l in ls -l and in stat output.
lrwxrwxrwx python -> python3.11 $ ln -s /path/to/target my-shortcut
If the target is deleted, the symlink breaks (dangling symlink). A hard link points to the data directly and stays valid even if the original is removed.
