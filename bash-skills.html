⌘ pwd Print working directory ▾
What it does
Shows your exact location in the filesystem as a full path.
Example
$ pwd /home/anazial/.openclaw/workspace
Tip
Use it whenever you feel lost. Think of it as "You Are Here" on a map.
⌘ ls List directory contents ▾
What it does
Shows all files and folders in your current directory.
Example
$ ls AGENTS.md BOOTSTRAP.md README.md
Common flags
-a -l -h -lah
⌘ cd Change directory ▾
What it does
Moves you into a different folder.
Examples
$ cd documents $ cd .. $ cd ~/.openclaw/workspace $ cd /var/www/html
Tip
cd .. goes up one level. cd ~ takes you home from anywhere.
⌘ hostname Show machine name ▾
What it does
Prints the name of the machine your shell is running on.
Example
$ hostname enid
Tip
Use it as a sanity check when jumping between your Mac and Enid via SSH.
⌘ whoami Show current user ▾
What it does
Prints the username you are currently logged in as.
Example
$ whoami anazial
Tip
Useful after sudo su to confirm whether you are root or your regular user.
Files & directories
⌘ cat Print file contents ▾
What it does
Dumps the entire file to the terminal at once. Best for short files.
Example
$ cat AGENTS.md
Tip
If your screen floods with text, switch to less instead.
⌘ less Scrollable file viewer ▾
What it does
Opens a file in a scrollable viewer. Better than cat for long files.
Example
$ less BOOTSTRAP.md
Controls
Arrow keys or spacebar to scroll. q to quit.
⌘ head Show first 10 lines ▾
What it does
Prints just the top of a file. Good for peeking.
Example
$ head AGENTS.md $ head -20 AGENTS.md
Common flags
-n
⌘ tail Show last 10 lines ▾
What it does
Prints the bottom of a file. Especially useful for log files.
Example
$ tail error.log $ tail -20 error.log
Common flags
-n
⌘ wc Word / line count ▾
What it does
Counts lines, words, or characters in a file.
Example
$ wc -lw AGENTS.md 42 310 AGENTS.md
Common flags
-l -w
⌘ file Identify file type ▾
What it does
Tells you what kind of file something is -- useful when the extension is missing or unclear.
Example
$ file image.png image.png: PNG image data, 1920 x 1080
⌘ find Search for files ▾
What it does
Searches for files by name, type, or other attributes. Searches recursively through subdirectories.
Examples
$ find ~ -name "AGENTS.md" $ find . -name "*.md" $ find / -name "nginx.conf"
With -exec
Run a command on every result. {} is the placeholder for each file found.
$ find . -name "*.rtf" -exec pandoc {} -o {}.md \;
Common flags
-name -exec -type
⌘ grep Search inside files ▾
What it does
Searches for text inside files. find looks for files, grep looks inside them.
Example
$ grep "session" AGENTS.md $ grep -r "session" ~/documents
Common flags
-r -i -n
⌘ mv Move or rename ▾
What it does
Moves a file or directory to a new location. Also used to rename things.
Examples
$ mv file.md ~/documents/ $ mv oldname.md newname.md $ mv folder/ ~/documents/ $ mv file.md .
Tip
Does not need -r for directories. The original is gone after moving.
⌘ cp Copy files ▾
What it does
Copies a file or directory. Original stays in place.
Examples
$ cp file.md ~/documents/ $ cp -r myfolder/ ~/documents/ $ cp *.md ~/documents/
Common flags
-r
⌘ rm Delete files ▾
What it does
Permanently deletes files or directories. No trash, no undo.
Examples
$ rm file.md $ rm -r myfolder/ $ rm -ri folder1/ folder2/
Common flags
-r -f -i
Warning
-rf deletes everything forcefully with zero confirmation. Double check before running.
⌘ mkdir Create directory ▾
What it does
Creates a new empty directory.
Example
$ mkdir newfolder $ mkdir ~/documents/projects
⌘ du Disk usage ▾
What it does
Shows how much disk space a directory is using.
Examples
$ du -sh myfolder/ $ du -sh * $ du -sh .
Common flags
-s -h
⌘ stat Read file metadata ▾
What it does
Reads everything the filesystem knows about a file -- permissions, size, owner, timestamps, inode number. Works on files, directories, symlinks, and devices.
Default output
$ stat myfile.md File: myfile.md Size: 1234 Blocks: 8 regular file Access: (0644/-rw-r--r--) Uid: (1000/anazial) Modify: 2026-05-20 09:00:00
Custom format (GNU / Ubuntu)
Use -c with format specifiers to print only what you need.
$ stat -c "%a %n" * 644 readme.md 755 myscript.sh $ stat -c "%a %A %U %G %n" myfile.md 644 -rw-r--r-- anazial anazial myfile.md
macOS difference
macOS uses BSD stat. Use -f instead of -c, and %Lp for octal permissions.
$ stat -f "%Lp %N" * # macOS only
Common flags
-c
⌘ pandoc Convert file formats ▾
What it does
Converts between document formats. Figures out the conversion from file extensions.
Examples
$ pandoc file.rtf -o file.md $ pandoc file.md -o file.docx
Batch convert with loop
$ for f in *.rtf; do pandoc "$f" -o "${f%.rtf}.md" done
Common flags
-o
Remote & SSH
⌘ ssh Connect to remote machine ▾
What it does
Opens a secure shell session on a remote machine. Everything you type runs there, not locally.
Examples
$ ssh anazial@100.x.x.x $ ssh anazial@enid
Tip
With Tailscale you can use the machine hostname instead of the IP address.
⌘ exit Leave SSH / switch user ▾
What it does
Ends the current shell session. Drops you back to your local terminal after SSH, or back to your regular user after sudo su.
Example
$ exit
Force disconnect
If the terminal is frozen, type ~. (tilde then period) to force close SSH.
⌘ scp Secure copy between machines ▾
What it does
Copies files between your local machine and a remote one over SSH. Run from your local machine.
Examples
$ scp file.md user@enid:~/ $ scp user@enid:~/file.md ~/Desktop/ $ scp -r folder/ user@enid:~/ $ scp user@enid:~/folder/* ~/Desktop/
Pattern
Remote paths use user@host:/path format. The colon separates machine from path.
Common flags
-r
⌘ tailscale status List Tailscale devices ▾
What it does
Shows all devices on your private Tailscale network with their IPs and hostnames.
Example
$ tailscale status
System & admin
⌘ sudo Run as administrator ▾
What it does
Superuser do. Temporarily runs one command with full admin privileges.
Examples
$ sudo apt install nginx $ sudo mv file.html /var/www/html/ $ sudo reboot
Tip
sudo su switches you fully into the root user. Use exit to get back out.
⌘ systemctl Manage background services ▾
What it does
Starts, stops, restarts, and checks the status of system services on Ubuntu.
Examples
$ sudo systemctl status nginx $ sudo systemctl start nginx $ sudo systemctl stop nginx $ sudo systemctl restart nginx
⌘ apt Install software (Ubuntu) ▾
What it does
Ubuntu's package manager. Installs, updates, and removes software.
Examples
$ sudo apt install nginx $ sudo apt update $ sudo apt update && sudo apt upgrade nginx
⌘ brew Install software (Mac) ▾
What it does
Homebrew -- Mac's package manager. Equivalent of apt on Ubuntu.
Examples
$ brew install helix $ brew install pandoc $ brew services start nginx
⌘ echo $SHELL Show current shell ▾
What it does
Prints which shell you are using. echo prints a value, $SHELL is an environment variable holding the shell path.
Example
$ echo $SHELL /bin/zsh (Mac) /bin/bash (Ubuntu)
⌘ sudo reboot Restart the machine ▾
What it does
Restarts the machine. If you are SSH'd in, your connection will drop immediately -- that is normal.
Example
$ sudo reboot
Tip
Wait 30 seconds then SSH back in as normal.

Path symbols
~
Home directory
▾
What it means
A shortcut for your home directory. Works anywhere a path is expected.
Examples
$ cd ~ $ ls ~/documents $ scp file.md user@enid:~/
.
Current directory
▾
What it means
Refers to wherever you currently are. Useful as a destination.
Examples
$ mv ~/downloads/file.md . $ cp ~/documents/* . $ du -sh .
..
Parent directory
▾
What it means
One level up from where you are.
Examples
$ cd .. $ cd ../../ $ mv file.md ../
/
Root / path separator
▾
What it means
At the start of a path, / means the absolute root of the filesystem. Between path parts, it separates directory names.
Examples
$ cd /var/www/html $ ls / $ find / -name "nginx.conf"
Tip
Paths starting with / are absolute (always from root). Paths without / are relative (from where you are).
Wildcards & patterns
*
Wildcard -- match anything
▾
What it means
Matches any characters. Use it to target groups of files by pattern.
Examples
$ ls *.md $ cp *.txt ~/backup/ $ scp user@enid:~/folder/* ~/Desktop/ $ find . -name "*.rtf"
Contents vs directory
folder/ copies the folder itself. folder/* copies what is inside it.
Operators & chaining
;
Run sequentially
▾
What it does
Runs commands one after the other, regardless of whether the first one succeeded.
Example
$ mkdir folder; cd folder
